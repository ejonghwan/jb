# 종류
- 중요
<element role="tablist">  - 탭 전체를 그룹핑
<element role="tab">  - 탭 하나하나를 칭함
<element role="tabpanel"> - 컨텐츠 하나하나를 칭함

- 대체가능
<element role="alertdialog"> -> dialog *근데 dialog 태그는 탭인덱스 쓸 수 없다는데 ....?
<element role="dialog"> -> dialog 
<element role="navigation"> -> nav 
<element role="complementary"> -> aside 
<element role="none"> -> div
<element role="region"> -> section 등 랜드마크 콘텐츠(만약 div를 섹션처럼 랜드마크콘텐츠로 만들려면 이 롤 지정)


* aria-live="prolite" ? 보조기기는 지금 읽고 있는걸 중단하지않고 기다렸다가 끝나면 읽어줌
* aria-live="assertive" ? 음성 낭독을 중지하고 이 메시지부터 바로 알려줌
* aria-atomic="true" ? 내용이 갱신되었을 때 부분적으로 갱신되도 영역을 처음부터 다시 읽어줌

# tooltip
- 초점을 받으면 설명을 팝업
- 툴팁은 자동으로 초점을 이동시키면 안됨(+초점을 받으면 안됨)
- 툴팁은 링크나 버튼이 있으면 툴팁이 아니라 dialog임 (컨텐츠 안에 초점을 받는 요소가 없어야함)
- ESC 또는 마우스를 빼면 사라져야 한다
- 여는 a/button에 aria-describedby="id reference list" 속성으로 연결 (2개 이상 가능)


# 예시
예) 연결시 초점이 a에 들어가면 자동으로 툴팁을 낭독기/보조기기에 전달하게됨 
- 인풋 툴팁 예시
<label for="tel">전화번호</label>
<input id="tel" type="tel" aria-describedby="TIP-TEL">
<p id="TIP-TEL" role="tooltip" hidden> 하이픈(-) 없이 숫자만 입력</p>

- 버튼 툴팁 예시
<button aria-describedby="TIP-DEL">게시물 삭제</button>
<p id="TIP-DEL" role="tooltip" hidden>게시물 삭제 후 복원할 수 없음</p>

# status
- 실시간 결과 정보. alert 만큼 중요하진 않지만 알림
- 이 요소를 갱신할 때 초점을 받지 않아야 한다
- 이 역할을 선언하면 자동으로 aria-live="prolite", aria-atomic="true" 속성이 할당됨 


# 예시

# 정의 
- HTML 요소로 불가능한 상태를 부여 

# 종류
<element aria-current=" page | step | location | date | time | true | false(defalut) ">  - a컨텐츠가 현재 페이지다(로케이션) 
<element aria-selected=" false | true | undefined(default) ">  - 다중선택가능한 곳에 선택된 상태
<element aria-haspopup=" true | menu | dialog | listbox | tree | grid | false(default) ">  - a가 팝업을 가지고 있다!
<element aria-expanded=" true | false | undefined(default) ">  - a가 가지고 있는 콘탠츠가 펼쳐져있다!  
<element aria-hidden=" true | false | undefined(default) ">  - 보조기기 숨기고 싶을 때
<element aria-invalid=" true | false(default) | grammar | spelling ">  - 검증 시
    page | step | location | date | time | true | false(defalut)
                                
# 설명
- aria-current   page | step | location | date | time | true | false(defalut)
- 현재 맥락과 일치하는 항목. 버튼이나 a에 많이 사용
1. page: 현재 페이지와 일치하는 시각적으로 강조한 링크
2. step: 현재 '단계'와 일치하는 시각적으로 강조한 링크
3. location: 플로우 차트에서 현재 '위치'와 일치하는 시각적으로 강조한 이미지 
4. date: 달력에서 현재 '날짜'와 일치하는 날짜
5. item: 시간표에서 현재 '시간'과 일치하는 시간 
6. true: 항목이 세트 내 현재 맥락과 일치함
7. false: 항목이 세트 내 현재 맥락과 일치하지 않음

# 설명
- aria-haspopup   true | menu | dialog | listbox | tree | grid | false(default)
- 대부분 a, button 요소에 적용하며 눌렀을 때 팝업이나 서브메뉴가 뜬다면 그런것들을 가지고 있다는 정보를 전달해주는 용도
- 팝업 유형은 menu, listbox, tree, grid, dialog로만 써야함 (대부분 menu와 dialog를 씀 )

1. true: menu와 동일한 의미
2. menu: menu(role) 팝업이 연결됨. menu(role)는 링크목록 a에 마우스 오버나 클릭 시 서브메뉴가 있다를 의미
3. dialog: dialog(role) 팝업이 연결됨. dialog(role)는 상호작용 요소(버튼 등)가 포함된 현재 문서의 하위창
