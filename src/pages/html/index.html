# HTML 문서 작성방법
1. HTML도 문서이기 때문에 개요의 연속임.
2. 책처럼 잘 섹션에 제목이 잘 정리되어 있어야 접근성/SEO에 적합
3. 구글크롬에 HeadingMap 이라는 익스텐션 사용해서 문서의 개요 확인
4. 헤딩태그들은 2회 이상 사용가능함. 
5. 다만 중요의 순서를 잘 정리해야됨
6. 타이틀은 문서의 제목이기에 한번만 쓸 수 있지만 헤딩은 섹션의 제목이기에 여러번 사용 가능
7. "섹셔닝루트"를 브라우저가 구현해놓지 않음. 따라서 (body blockquote details dialog figure fieldset td...)끼리도 헤딩을 공유함.
 - 섹셔닝루트란 명세에서 위 태그들은 섹션의 범위에서 제외되지만..브라우저는 구현하지않아서 그냥 통합

# 시멘틱 태그 
<div> <span> - 아무 의미 없는 태그(최대한 지양)

1. Sectioning 
- 섹셔닝은 아니지만 함께 쓰는 요소
<hx> 
<hgroup> 
<header> - 도입부, 헤딩, 헤딩그룹, 목차, 검색, 로고 (복수가능, 생략가능)
<footer> - 저자, 저작권, 연락처, 관련문서 (복수가능, 생략가능)

- 섹셔닝 요소 (헤딩 태그와 같이 쓰는걸 강력권장)
<article> - 독립적 배포 가능한 완결형 콘텐츠. 뉴스기사, 블로그 본문, 댓글
<aside> - 페이지의 주요 내용이 아닌..(배너, 연관 콘텐츠)
<nav> - 사이트의 주된 탐색 메뉴
<section> - 주제별로 나누거나 묶는

2. <main>
- 페이지에 딱 1개만 가능(여러개를 쓸 때는 하나만 디스플레이해야함)
- 섹셔닝 콘텐츠의 자식요소로 올 수 없음 
- body, div 외 자식요소로 올 수 없음

3. <dialog>
- 팝업 등에 활용
- 뜨는 순간 포커스가 여기로 와야함. 
- 탭(포커스)이 팝업안에서만 돌아야됨

4. <figure>
- 문서의 주된 흐름을 위해 참조하는(부록으로 옮겨도 괜찮은) 독립적인 완결형 요소. 이미지, 도표, 코드등 
- 내용물과 설명(<figcaption>)을 포함..단한번만 사용 가능
- 선택적으로 처음 또는 마지막에 <figcaption> 요소 포함 & 생략 

5. <mark>
- 독자의 주의를 끌기 위한 하이라이트 
- 현재 독자의 행위나 관심에 따라 강조한 것 
예) 검색결과 목록에서 사용자가 입력한 키워드

6. <address>
- 집주소 연락처 같은게 아닌, 저작권 정보..저자의 정보를 나타낼 때 사용
예) footer
      addres 
        p 이종환 / jjong@naver.com / 000-0000-0000


7. 정보 추가 삭제
<ins> 추가한 내용
<del> 삭제한 내용
- 둘 다 투명요소
- 여러 단락을 한번에 포함하는 것은 부적절


8. <progress value="0.5">
- 로딩바

9. 기타 현재 특별한 의미로 남아있는 예전 요소들 
<b> 강조할 의도 없는 스타일만 볼드  -> <strong> 
<i> 현재 맥락과 다른 분위기 -> <em>
<s> 정확하지 않거나 관련없는 -> <del>
<u> 오타, 중국의 고유명사 등의 표기 -> <ins>

# 로딩속도의 이해
우리가 보는 모니터는 대부분 60fps (모니터가 1초동안 60번을 깜빡이면서 화면을 그림)
1초는 1000ms 이므로 60/1000을 하면 16.66666....ms 
즉 모니터가 1번 깜빡이는 속도가 16ms 이므로 이 안에 로딩을 끝내야 덜컥거림이 안생김
